## Mensagem 4 — Erro ao compilar o devcontainer (2ª tentativa)

**Usuário:**
> Anexou novo log de erro com a mesma imagem revertida.

**Erro:**
```
ERROR: failed to resolve source metadata for mcr.microsoft.com/devcontainers/python:3-3.14-trixie:
failed to do request: Head "https://mcr.microsoft.com/...": EOF
```

**Assistente:**
Confirmou que o problema é de conectividade com `mcr.microsoft.com` (Microsoft Container Registry), não das tags.

**Solução aplicada:**
- Criou `.devcontainer/Dockerfile` usando `python:3.12-slim` do **Docker Hub** (endpoint diferente):

```dockerfile
FROM python:3.12-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    git curl sudo \
    && rm -rf /var/lib/apt/lists/*

ARG USERNAME=vscode
ARG USER_UID=1000
ARG USER_GID=${USER_UID}

RUN groupadd --gid ${USER_GID} ${USERNAME} \
    && useradd --uid ${USER_UID} --gid ${USER_GID} -m ${USERNAME} \
    && echo "${USERNAME} ALL=(root) NOPASSWD:ALL" > /etc/sudoers.d/${USERNAME} \
    && chmod 0440 /etc/sudoers.d/${USERNAME}

USER ${USERNAME}
WORKDIR /home/${USERNAME}
```

- Atualizou `devcontainer.json` para usar `"build": { "dockerfile": "Dockerfile" }` em vez de `"image"`
