## Mensagem 2

**Usuário:**

Compartilhou o arquivo `logs/errors.log` com os seguintes erros:

```log
WARNING 2026-05-22 20:29:23,046 views Erro ao processar resposta:
Estrutura de resposta inesperada: expected string or bytes-like object, got 'NoneType'.
Corpo: {'choices': [{'finish_reason': 'length', 'message': {'role': 'assistant', 'content': None}}], ...}

ERROR 2026-05-22 20:29:23,048 Service Unavailable: /send/
ERROR 2026-05-22 20:29:23,048 "POST /send/ HTTP/1.1" 503 61
```

> Corrija estes erros

---

**Assistente:**

Analisou `services.py` e `views.py`. Identificou duas causas:

1. **`content: None`** — o modelo `nvidia/nemotron-nano-9b-v2:free` esgotava todos os tokens com raciocínio interno (`reasoning_tokens: 351 > max_tokens: 300`), retornando `content: None`. O `re.sub` falhava ao receber `None`.

2. **`max_tokens` insuficiente** — 300 tokens para resposta normal e 800 para extração eram pouco para modelos com raciocínio.

**Correção em `chat/services.py` — método `_handle_response`:**

```python
choice = choices[0]
content = choice["message"]["content"]
if content is None:
    finish_reason = choice.get("finish_reason", "unknown")
    raise ServiceUnavailableError(
        f"Modelo retornou content=None (finish_reason={finish_reason}). "
        "Possivelmente esgotou tokens com raciocínio interno."
    )
```

`ServiceUnavailableError` aciona o retry automático do cliente.

**Correção em `chat/services.py` — `max_tokens` aumentado:**

```python
max_tokens = 1200 if is_extraction else 600
```
